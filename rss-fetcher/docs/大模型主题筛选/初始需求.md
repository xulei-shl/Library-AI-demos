src\core\book_vectorization\excel_exporter.py，执行图书检索后导出的书目完整元数据，还需要调用大模型根据用户输入的文章主题分析报告，对每本书目进行主题相关性和维度契合度的评估。

- 大模型调用的逻辑请直接复用 src\utils\llm。大模型配置见 config\llm.yaml
- 系统提示词为 prompts\主题筛选.md（默认也是使用langfuse同名的提示词）
- 用户提示词包括2个部分：
    - 文章分析报告，是一个md文档，完整提取作为用户提示词的组成部分。md文档可以在命令行窗口中人工输入
    - 图书元数据：从excel中提取每一行的书目元数据，作为用户提示词的组成部分。excel文件也是命令行窗口中人工输入
        - 书目条码id（书目条码列）
        - 书名（豆瓣书名列）
        - 副标题（豆瓣副标题列）
        - 作者（豆瓣作者列）
        - 丛书（豆瓣丛书列）
        - 内容简介（豆瓣内容简介）
        - 作者简介（豆瓣作者简介）
        - 目录（豆瓣目录）
- 大模型返回的结果json格式保存到excel新增列中，并将各个字段解析到各自的新增列中
- 也有一个新增列记录大模型的状态，并与失败或者json解析错误的数据兜底重试。也就是在全部数据llm处理完后，再进行一次兜底重试对失败的数据再次调用llm进行处理。
- 交互方式，在` python scripts/retrieve_books.py --interactive`中新增命令5 执行这个大模型评选模块，用户需要输入文章分析报告的路径，以及图书元数据的excel文件路径。
