# 故障复盘报告
- **日期**: 2025-12-22
- **严重级别**: High

## 1. 问题现象
主页地图底图不可见，用户只看到纯色背景；控制台仅出现数据加载日志后被清空，没有瓦片网络请求成功记录。

## 2. 根本原因 (Root Cause)
底图使用 StadiaMaps Toner 图层，但请求中未携带 API Key。StadiaMaps 自 2023 年起要求授权密钥，未授权的瓦片请求全部返回 401 Unauthorized，导致地图底图完全无法加载。

## 3. 解决方案
将底图切换为无需密钥的 OpenStreetMap (ol/source/OSM) 并开启 crossOrigin: 'anonymous'，确保瓦片可直接加载。无需额外配置即可恢复底图显示。

## 4. 预防措施
- 如需第三方底图，必须在文档中列明 API Key 配置方式并在代码中读取环境变量。
- 开发调试时关注 Network 面板中瓦片请求状态，发现 4xx/5xx 应立即排查。
- 在验证清单中新增“底图授权配置”检查项。
